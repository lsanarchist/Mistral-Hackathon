# Project Context: Mistral-Hackathon

## Overview
This document provides a comprehensive overview of the Mistral-Hackathon project, detailing its structure, components, and functionality. It is designed to help AI bots understand the project's architecture, codebase, and purpose.

## Project Structure

### Root Directory
- **bin/**: Contains the `triageprof` binary.
- **cmd/**: Contains the main application entry point.
- **demo-output/**: Example output files generated by the project.
- **examples/**: Demo servers and scripts for various languages (Go, Node.js, Python, Ruby).
- **internal/**: Core project code, organized into subdirectories for different functionalities.
- **out-demo/**, **out-missing/**, **out-node/**, **out-python/**, **out-ruby/**: Output directories for different demo scenarios.
- **plugins/**: Plugin system for extending functionality.
- **test-llm-output/**: Test output files.
- **web/**: Web interface and dashboard files.

### Key Files
- **go.mod**, **go.sum**: Go module files for dependency management.
- **demo.sh**, **ruby-load.sh**: Scripts for running demos.
- **connection-quality-alerts.json**, **performance-alerts.json**: Example alert configurations.
- **.gitignore**: Specifies files and directories to ignore in Git.
- **AGENTS.md**: Documentation about agents used in the project.
- **COMPASS.md**: Project compass and guidelines.

## Key Functionality

### Authentication
- The `internal/auth/auth.go` module handles authentication, ensuring secure access to the application's features.

### Analysis
- The `internal/analyzer/analyzer.go` module performs core analysis tasks, processing data to generate insights.

### Pipeline
- The `internal/core/pipeline.go` module defines the processing pipeline, orchestrating the flow of data through various stages of analysis and reporting.

### LLM Integration
- The `internal/llm/` module integrates with Large Language Models (LLMs) to generate insights, handle prompts, and manage caching. It supports multiple providers like Mistral and OpenAI.

### Model Definitions
- The `internal/model/` module defines the data models used throughout the project, including insights and reports.

### Plugin System
- The `internal/plugin/` module provides a plugin system for extending the project's functionality. It includes JSON-RPC communication and manifest handling.

### Reporting
- The `internal/report/` module generates reports based on the analysis performed by other modules.

### Web Server
- The `internal/webserver/` module implements a web server for the project, providing a user interface and API endpoints for interaction.

### Web Interface
- The `web/` directory contains the web interface files, including HTML, CSS, and JavaScript for the dashboard and plugin management.

### Examples and Demos
- The `examples/` directory contains demo servers and scripts for various languages, showcasing the project's capabilities.

## Build and Run

### Prerequisites
- Go (for building the main application and plugins)
- Node.js (for running the Node.js demo server)
- Python (for running the Python demo server)
- Ruby (for running the Ruby demo server)

### Building
1. Navigate to the project root directory.
2. Run `go build -o bin/triageprof cmd/triageprof/main.go` to build the main binary.
3. Build plugins as needed using the respective build commands for each language.

### Running
1. Start the main application using `bin/triageprof`.
2. Run demo servers using the scripts in the `examples/` directory.
3. Access the web interface by navigating to the appropriate URL in a web browser.

## Testing

### Unit Tests
- Run `go test ./...` to execute all unit tests.

### Performance Tests

### Integration Tests
